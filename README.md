Домашка 8. Попытка сделать независимым от конкретной реализации участников предоление ими препятствий
ЛОГИКА:
Перечисление ObstacleEnum, задает типы препятствий как для участника (те, которые он может преодолевать) и для самих препятствий(что они из себя предствляют)
Класс Competitor. Получает массив участников и массив препятствий. Метод startCompetition запускает соревнования
Интерфейс Participant. Это интерфейс участника.
Классы Human, Cat и  Robot, реализуют этот интерфейс. Здесь код дублируется, потому что согласно заданию, нельзя эти классы наследовать от одного класса. Если бы это было бы запрещено, то вообще можно было обойтись одним общим предком.
Интерфейс Obstacle - это интерфейс препятствия.
Класс ObstacleReal - это реализация интерфейса Obstacle, можно было обойтись и без интерфейса, но по занятию проходим интерфейсы, вот пихаем их везде.
ЛОгику каждого класса и интерфейса смотреть в комментариях
